## 2. Exit-gate evidence

- **Classifies known-solvable and known-unsolvable fixtures:** `Solvability.test.ts` classifies the corridor and single-gap fixtures `solvable` (each with a witness tape that, replayed with no agent, reaches the goal), and the two-jump and walled-goal fixtures `unsolvable`, with zero misclassification. The two-jump case is the axiom's teeth: the search never obtains a second jump because the engine's lock never grants one (`world.jumpLock` is read, never re-implemented — dm-0011/0020).
- **Optimization delta computed on fixtures:** `Optimization.test.ts` computes the five tiers and delta on the gap fixture (positive delta, not rejected) and the flat fixture (delta 0, rejected per REQ-102), and cross-checks the authored par times.
- **Five agent archetypes drive the deterministic sim headlessly:** `AgentArchetypes.test.ts` runs all five (First-Time, Cautious, Experienced, Expert-Speedrunner, Curious-Explorer) to completion, proving they are five *behaviors* (pairwise-distinct tapes) and that every run is bit-reproducible and tape-replayable.
- **Two isolated validation passes (REQ-140):** Local (`src/eval/local/*`, per-level) and Macro (`src/eval/macro/*`, per-chapter) are separate modules with separate verdict types; the macro pass consumes local verdicts as data and runs no simulation — enforced by an import-isolation scan test.
- **Determinism contract upheld:** every archetype and every audit is deterministic; live-run ≡ tape-replay bit-equality holds per archetype; no `Math.random`, no delta-time, no transcendental function under `src/eval/` (grep-audited like `src/systems/`).

## 5. Invariant compliance notes

- **Determinism (dm-0017 audit extended to `src/eval/`):** the `EvalIsolation` scan confirms **no transcendental function** and no `Math.random` under `src/eval/` — only `+ − × ÷ %`, `Math.sqrt/min/max/abs/floor/ceil/trunc`. Agent decisions and search quantization are whitelist-math only, so tapes replay bit-identically across JS engines.
- **One-way dependency (dm-0022):** a scan test fails the build if any module under `src/{core,systems,components,entities,schema}` imports from `src/eval/` — the simulation never knows it is being judged.
- **The axiom is ground truth, never a parameter (dm-0011/0020):** no `maxJumps`-like knob exists in any eval module; solvability reads `world.jumpLock` and lets the engine's lock enforce the one jump — the two-jump fixture is unsolvable *by construction*.
- **Agent RNG ≠ sim RNG (dm-0024):** agents draw only from their own threaded stream; `state.rng` belongs to the simulation, so tape replay (no agent) reproduces the live run bit for bit — asserted per archetype.
- **Replay-tape serialization (dm-0023):** owned here at last (open since P2), versioned from day one, with the full P2 strict-JSON discipline (canonical order, unknown-key rejection, Result-typed parse).
- **Content gate:** no campaign content generated; every fixture is in-code unit scaffolding or an existing `test/fixtures/` file. The M2 hard gate is untouched — it stays closed until P5 and P6 verify.
