## 2. Exit-gate evidence

- **Macro state updates deterministically from a fixture campaign:** `CampaignDirector.test.ts`'s determinism test runs `processCampaign` twice over the same `ChapterRecord[]` and asserts a deep-equal (not merely `==`) `CampaignReport`; `updateState` has its own twice-called deep-equal test plus a JSON-snapshot proof that it never mutates its input state.
- **Flags a synthetic difficulty spike:** a synthetic 3-chapter fixture campaign (healthy → unhealthy → healthy) produces a `CampaignAlert` attached to exactly the middle chapter's `ChapterHealthReport.alerts` and to `CampaignReport.alerts`, and to no other chapter. A second test proves the threshold is profile data, not a literal: raising `spikeDropThreshold` on the identical fixture suppresses the same spike.
- **Every one of the ten REQ-031 macro variables is genuinely computed, not just typed:** `knowledgeState` (`KnowledgeModel`), `behaviorState` (`TapeAnalyzer`), `emotionalState`/`skillCurve`/`curiosityTrend`/`optimizationDepth` (`CampaignDirector`), `mechanicsIntroduced`/`mechanicsMastered` (`MechanicTracker`), `chapterHealth` (`ChapterHealth`), `retentionPrediction` (`CampaignDirector`) — each has its own dedicated test file proving the computation, not just the shape.
- **P4/P5 outputs consumed by reference, never re-derived (dm-0045/dm-0046):** `MechanicTracker` takes `CoverageMatrix` as a parameter and never calls `DesignSpace.coverageMatrix()`; `ChapterHealth` passes `MacroVerdict`'s four criteria through verbatim (proven by an exact-object test using a distinct findings array); `CampaignDirector`'s `emotionalState` rolls GDOS's own `'emotional-threshold'` gate scores rather than a re-derived proxy.
- **Zero P8 dependency (dm-0044):** `TapeAnalyzer.analyzeTape` depends only on `ArchetypeRun` (a P4 harness output) and `CampaignProfile`; no `campaign/` file imports anything under `src/tools/` or a telemetry module (none exists yet, and none is needed).
- **Calibration is external (dm-0031/dm-0045, carried into P6):** `campaign/` gate/director logic contains zero calibration literals — every threshold, weight, and rate reaches code through `CampaignProfile`. Nine independent two-profile tests (spread across all six test files) each score or classify one identical fixture under two profiles and assert the outcome flips while the fixture is unchanged.
- **`campaign/` is isolated and never re-audits (dm-0047, now structurally enforced — dm-0053):** four scans in `EvalIsolation.test.ts` prove `src/eval/campaign/` imports no `AgentHarness`/`AgentPolicy`/`Archetypes`/`Evaluate`/`eval/local/`/`core/Engine`, no `gdos/` gate-internal module (`Emotional`/`Streamability`/`InfoDensity`/`Curation`/`Cdre`/`DesignMemory`/`Judge`/`Profile`), and imports `MacroVerdict` as a type only. The math whitelist is scan-enforced across `src/eval/campaign/` (closing a real gap: the parent `src/eval/` scan does not recurse into subdirectories, so `campaign/` was previously uncovered by any math-whitelist test until this phase closed it).
- **Determinism:** every function in `campaign/` is pure — no `Math.random`, no `Date.now`, no delta-time; whitelist math only (`Math.abs`, comparisons, `+`, `÷`), scan-enforced.
