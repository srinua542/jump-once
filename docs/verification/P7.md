## 2. Exit-gate evidence

- **The pipeline produces a schema-valid level that passes P4 solvability and P5 gates:** `GenPipeline.test.ts`'s IRD-exit test manufactures a default concept end-to-end and asserts the emitted `PipelineProduct.def` re-proves through `parseLevel`, is `solvable` (S4.2), is `jump-necessary` (S7.2 counterfactual — the REQ-012 load-bearing check), and its `GdosReport.pass` is true. The product carries everything a campaign `LevelRecord` needs (representative run, `mechanicsExercised`, provenance).
- **…or is correctly rejected+deleted with a logged reason:** four poisoned-concept tests each land a rejection at a DISTINCT correct phase (`concept`, `structural-prototyping`, `ai-council`, `sign-off`), each carrying a typed `phase` + `reason` + `findings`; a fifth test asserts a rejection persists no product (`'accepted' in outcome` is false) — "deletion" means only the rejection record survives (dm-0059).
- **The one jump is proven load-bearing (REQ-012, dm-0041 discharged):** the S7.2 counterfactual audit runs an exhaustive no-jump reachability search; a level completable without the jump is killed as `jump-irrelevant` with a replayable witness — the necessary-but-insufficient tape proxy could not detect this. The pipeline's phase 3 gates on `jump-necessary`, so no accepted product is completable without its jump.
- **Calibration is external (dm-0031/dm-0057, extended to `gen/`):** `gen/` generation logic contains zero calibration literals — every threshold, weight, budget, and entity-dynamics value reaches code through a parsed `GenProfile` (its own third versioned schema, separate from `ScoringProfile` and `CampaignProfile`). Two-profile externalization proofs cover every calibrated field across `GenProfile.test.ts`, `GenLifecycle`, `GenPda`, `GenGenerator`, `GenCreativity`, `GenIntentGate`.
- **Generation is deterministic (dm-0059/dm-0062):** every `gen/` function is a pure function of its inputs — all stochastic steps thread a `core/Rng` state, all loops are structurally bounded (hard cap + epsilon), no `Math.random`, no `Date.now`. `generateCandidate`, `evolveLevel`, and `manufactureLevel` each have a two-runs-identical determinism test.
- **`gen/` is isolated and consumes evaluation only through public seams (dm-0057):** `GenIsolation.test.ts` proves nothing outside `gen/` imports `gen/`; `gen/` imports no engine/harness/policy/archetype/system module, no `gdos/` gate internal (`Emotional`/`Streamability`/`InfoDensity`/`Score`/`Economy`/`Curation`), and reads `campaign/` records as types only; the math whitelist and the no-clock ban are scan-enforced across `src/gen/`.
- **The lifecycle registry blocks exhausted mechanics end-to-end (REQ-082):** `isBlocked` is enforced at BOTH the concept boundary (`validateConcept`, proven in `GenConcept.test.ts` and the pipeline's concept-phase rejection test) and the PDA (`GenPda.test.ts` — a blocked mechanic never appears in an opportunity, and the drop is counted).
- **No content authored:** every `LevelDefinition` P7 emits exists only inside tests (generated fixtures); there is no `levels/` directory and no chapter authoring — the content gate holds through P7 (P10 authors real content, behind M3 = P7 + P8).
